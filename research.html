Japan Walk Guide

Japan Walk Guide is a navigation tool, assisting individuals with mobility challenges navigating urban Japan. There is a big need to help those with accessibility issues in dense urban environments like Tokyo. This web app uses real-time data, and crowd-sourced information and many accessibility features.

There is also walking route assistance and surrounding information as you can see in the diagram, with detailed station information.

This app has accessibility-focused design, with color-coded icons, voice-assisted navigation for hands-free use catering to a diverse portfolio of users. Community-driven updates ensure changes in the environment and reflect real-time, and the app remains up to date at all times. Additionally, the app has offline functionality and works on a variety of platforms. It can be easily launched by scanning a QR code.

The app also operates on a similar paradigm of easy access; Users would not download a dedicated app due to psychological barriers of installation, so our embedded widget is very suitable in this scenario.

What we can learn:

  • User-centered Design: Japan Walk shows the importance of designing with the users in mind. We can adopt a similar approach using a set of design heuristics like a checklist, including visual, auditory, and mobility impairments.
  • Community Platform: Japan Walk relies on user contributions to stay updated, so changes are reflected real-time. We could potentially implement a similar model, allowing users to submit updates.
  • Route Guidance: The app incorporates route guidance and detailed information on locations and congestion, this is very useful and we should think about incorporating a route guidance algorithm.

Wheelmap

Wheelmap.org allows users to search and view the accessibility status of public places (e.g., restaurants, shops, and public buildings) using a color-coded system (green = fully accessible, yellow = partially accessible, red = not accessible).

This is a global platform, delivering reliable and concise information on accessibility.

What we can learn:

  • Simple Intuitive Interface: Wheelmap has a clean and straightforward interface, we can prioritize simplicity in our design so it's easier to navigate and understand information.
  • Color-coded System: Color-coded visual cues, and pleasing icon graphics can provide understandable cues on the accessibility status of a location. We will implement this sort of visual cue, to make our UI more intuitive.
  • Lacks Necessary Accessibility Features: Wheelmap is effective for most people however there is no voice assistance for hands-free, or UI design options for those who have trouble seeing. We can incorporate these to make our product inclusive and user-friendly for those with diverse needs.
